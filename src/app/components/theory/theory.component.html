Angular Complete Tutorial

1.Angular Basics

1.What is Angular?

Angular is a JavaScript based client-side development framework which is used to build the Single Page Applications (SPA) for Mobile and Desktop.

2.What is a Framework?

A framework is a platform for developing Software Applications.

A framework can have pre-defined classes and functions that can be re-used to add several functionalities, which otherwise we would have to write it from scrach by our own.

3.What is a Single Page Application (SPA)?

A Single Page Application (SPA) is a web application, which has only one HTML page. When we navigate around, only the content of the HTML page changes. The page itself never changes.

4.What is the command to check the node version?

Command : node -v

5.What is the command to check the npm version?

Command : npm -v

6.What is the command to install Angular CLI?

Command : npm i -g @angular/cli@latest

6.What is the command to check the Angular version?

Command : ng v

7.What is the command to create a new angular project?

Command : ng new <Project_Name>

8.What is the command to run the angular application?

Command : ng serve

9.What is the default port number to run the Angular Application?

port : 4200

 2.File and Folder structure

node_modules :

The node_modules folder contains all the third party libraries which are used to build our Angular Application.

package.json

The package.json file contains all the list of third party dependencies which are helpfull to build and run the Angular application.

.editorconfig

The .editorconfig file is used to define the environment coding standards which are followed by all developers.

.gitignore

The .gitignore file is used to listout the files and folders which we want to exclude while code checkin onto the github.

angular.json

The angular.json file is used to define the entire application configurations, like entry point, index, styles, scripts, etc.

package-lock.json

The package-lock.json file records the exact version of every installed dependency with it's version and sub dependency along with its version. The package-lock.json file is responsible to install same versions of dependencies throughout all environments.

tsconfig.json

The tsconfig.json file contains bunch of settings for typescript compiler, based on this settings the typescript compiler compiles typescript code JavaScript code which browser can understand.

index.html

The index.html is the main html file, which was render on to the browser.

main.ts

The main.ts file which is the starting point of the angular application, the bootstrapping process of the angular application will start at this file.

styles.css

The styles.css file is used to define the global styles of the angular application.

What is Webpack?

The webpack is used to bundle the JavaScript files and other files into one or more JavaScript bundles.

What are the files injected into the index.html while building the Angular Application?

runtime.js ----- webpack runtime file.

polyfills.js ----- polyfills scripts which are supported to verity of modern browsers.

main.js ----- The main.js is the code of the application.

vendor.js ----- The vendor.js contains scripts from the angular/core library and all the third party libraries which are helpfull to build and run the angular application.

styles.js ----- It contains all the styles of the angular application.

***What is the Difference between TypeScript and JavaScript?

TypeScript is a strongly typed programming language but JavaScript is a dynamically typed programming language.

TypeScript have few Object oriented features which we donot have in JavaScript like interfaces, access modifiers, generics, etc.

In TypeScript we can catch errors at compile time insted of runtime.
